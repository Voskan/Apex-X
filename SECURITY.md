# Security Policy

## Supported Versions
This repository is pre-1.0. Security fixes are applied on the default branch.

## Reporting a Vulnerability
Report vulnerabilities privately by contacting project maintainers.
Include:
- affected component
- reproduction steps
- impact assessment
- suggested remediation (if available)

Do not open a public issue for unpatched vulnerabilities.

## Response Process
- Acknowledge report within 3 business days.
- Triage severity and affected scope.
- Publish a fix and coordinated disclosure note when ready.
